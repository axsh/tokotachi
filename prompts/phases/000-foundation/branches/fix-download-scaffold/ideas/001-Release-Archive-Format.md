# 仕様書: リリースターゲットアーカイブのバージョンディレクトリ化

## 1. 背景 (Background)

現在、ツールリリース用ビルドスクリプト `scripts/dist/tool/internal/package.sh` によって生成されるリリース用アーカイブ（`tar.gz` や `zip`）は、中間ディレクトリを含まずにバイナリやインストーラ等のファイル群をルートにフラット配置して圧縮しています。
このため、ユーザーがアーカイブを展開した際、カレントディレクトリにすべてのファイルが直接展開されてしまい、ディレクトリの整理や管理が困難になる課題があります。

展開時に自動的にバージョン番号を含むフォルダ（例: `tt-v1.0.0/`）が作成され、その中にファイルが展開されるようにパッケージング方式を変更します。

---

## 2. 要件 (Requirements)

*   リリースアーカイブを展開した際、ツールIDとバージョン番号を含むディレクトリ（例: `{TOOL_ID}-{VERSION}`、具体的な値として `tt-v1.0.0` など）が作成され、その配下にすべてのファイル（バイナリ、インストーラ等）が展開されること。
*   対象アーカイブ形式は、`tar.gz` および `zip`（Windows targets）の両方とする。
*   既存のビルドプロセスや、生成されるチェックサム生成処理（`checksums.txt`）に悪影響を与えないこと。

---

## 3. 実現方針 (Implementation Approach)

*   **変更対象ファイル**: `scripts/dist/tool/internal/package.sh` ([package.sh](file://scripts/dist/tool/internal/package.sh))
*   **変更内容**:
    *   アーカイブ圧縮用の一時ディレクトリ（`tmp_dir`）の中に、さらに `{TOOL_ID}-{VERSION}` 形式のバージョン付き中間ディレクトリ（`pkg_dir_name`）を作成する。
    *   バイナリおよび Windows 用インストーラファイル群を、直接 `tmp_dir` にコピーするのではなく、この中間ディレクトリ（`pkg_dir`）配下にコピーする。
    *   圧縮処理を行う際、`tmp_dir` を基準ディレクトリ（CWD）とし、中間ディレクトリ `pkg_dir_name` 自体を圧縮対象としてアーカイブを作成する。
        *   `zip` コマンドを使用する場合、ディレクトリ再帰指定（`-r` フラグ）を追加する。
        *   PowerShell の `Compress-Archive` を使用するフォールバック処理でも、ラッパーディレクトリの中身を全て圧縮することで中間ディレクトリを含める。

---

## 4. 検証シナリオ (Verification Scenarios)

1.  **アーカイブ作成と展開の検証**:
    *   `scripts/dist/tool/release.sh tt v1.0.0` （または対象ツール）を実行してリリース用アーカイブを生成する。
    *   生成された `.zip` や `.tar.gz` を一時ディレクトリに展開する。
    *   展開されたファイル群が `tt-v1.0.0/` ディレクトリの中に正しく収まっていることを確認する。

---

## 5. テスト項目 (Testing for the Requirements)

リリース生成処理をカバーする自動テストを実行します。

### ビルド・全体検証

1.  **ビルド＋単体テストの実行**:
    ```bash
    scripts/process/build.sh
    ```
2.  **リリース自動テスト、または関連するビルドスクリプト実行**:
    リリースタスク用のビルドスクリプト単体またはビルド検証を行います。
    ```bash
    scripts/process/build.sh --backend-only
    ```
