# Release Notes

## What's New

[New]

- Release tags in the `vX.Y.Z` format (e.g., `v0.4.4`) are now added on the same commit as legacy tags (e.g., `tt-v0.4.4`), enabling external Go projects to refer to specific versions using commands like `go get github.com/axsh/tokotachi@v0.4.4` with proper semantic versioning.
- Version selection and fetching through Go module proxies (`proxy.golang.org`) with the `vX.Y.Z` tag format is now fully supported.

[Changed]

- The GitHub release tag format is unified to `vX.Y.Z` (without the tool ID prefix).
- The `publish.sh` script now generates tags using only the version (e.g., `v0.4.4`), instead of including the tool ID.
- The tag extraction logic in `github-upload.sh` has been updated to target all tags starting with `v` and follow the `vX.Y.Z` format, with no further pre-processing steps.
- The LLM prompt in `features/release-note/internal/summarizer/summarizer.go` and generated release notes have switched from Japanese to English, and now use English headings ([New], [Changed], [Removed], etc.).
